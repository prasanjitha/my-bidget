## Story REP-001: Generate PDF Report

**Story ID**: REP-001  
**Story Title**: Create Monthly Budget Report PDF  
**Priority**: Should Have (P1)  
**Story Points**: 8

### User Story
**As a** user  
**I want to** generate a PDF report of my budget and expenses  
**So that** I can save, print, or share my financial summary

### Acceptance Criteria

**Given** the user is on the Settings page  
**When** the Export Monthly Report card is displayed  
**Then** it should show:
- PDF/document icon
- Label: "Export Monthly Report"
- Subtitle: "Generate and save PDF of current cycle expenses"
- GENERATE button

**Given** the user taps the GENERATE button  
**When** the button is pressed  
**Then** a loading indicator should display  
**And** PDF generation should begin

**Given** the PDF is being generated  
**When** the process runs  
**Then** the PDF should include the following sections:

1. **Header**
   - Title: "Bidget - Monthly Report"
   - Logo or app icon (optional)

2. **User Information**
   - Name: [User's name]
   - Budget Cycle: [Start Date] - [End Date]
   - Report Generated: [Date and Time]
   - Currency: [Selected Currency]

3. **Summary Section**
   - Allocated Budget: [Amount]
   - Total Spent: [Amount]
   - Remaining Balance: [Amount]
   - Status: "In Budget" or "Over Budget" (with visual indicator)

4. **Category Breakdown Table**
   - Columns: Category | Allocated | Spent | Remaining
   - Rows: Each category with amounts
   - Total row at bottom

5. **Detailed Expenses Table**
   - Grouped by date
   - Columns: Date | Title | Category | Amount
   - Each date group with subtotal

6. **Footer**
   - Report generated by Bidget app
   - Generated on: [Date and Time]

**Given** PDF generation succeeds  
**When** the PDF is created  
**Then** it should be saved to device downloads folder  
**And** file name format: "Bidget_Report_[YYYY-MM-DD].pdf"  
**And** a success message should display: "Report saved to Downloads"  
**And** a "Share" button should appear

**Given** PDF generation fails  
**When** an error occurs  
**Then** an error message should display: "Unable to generate report. Please try again."  
**And** allow user to retry

**Given** there are no expenses in the cycle  
**When** PDF is generated  
**Then** the report should still be created  
**And** show zero values  
**And** empty expense table

**Given** the user has permission to write to storage  
**When** PDF is saved  
**Then** save to: `/storage/emulated/0/Download/` (Android)

**Given** the user denies storage permission  
**When** PDF save is attempted  
**Then** request permission again  
**Or** display error: "Storage permission required to save report"

### Technical Notes
- Use `pdf` package for Flutter
- Fetch data:
  - User info from Firestore `users` collection
  - Expenses from `expenses` filtered by current budgetCycle
  - Category budgets from `categoryBudgets`
- PDF styling:
  - Font: Roboto or default
  - Colors: Brand colors
  - Tables: Bordered, alternating row colors
  - Page size: A4
  - Margins: 20mm
- File operations: Use `path_provider` package
- Permissions: Use `permission_handler` package
- Progress indicator: Show percentage if possible

### Dependencies
- SET-001 (Settings page structure)
- HOME-005 (Total spend calculation)
- SUM-002 (Category breakdown data)
- CYC-002 (Current budget cycle)

## Story REP-002: Share PDF Report

**Story ID**: REP-002  
**Story Title**: Share Generated PDF via System Share  
**Priority**: Should Have (P1)  
**Story Points**: 3

### User Story
**As a** user  
**I want to** share my PDF report  
**So that** I can send it via email, messaging apps, or cloud storage

### Acceptance Criteria

**Given** a PDF report has been generated  
**When** the success message is displayed  
**Then** a "Share" button should be visible

**Given** the user taps the Share button  
**When** the button is pressed  
**Then** the system share sheet should open  
**And** the PDF file should be attached  
**And** sharing options should include:
- Email
- Messaging apps (WhatsApp, Telegram, etc.)
- Cloud storage (Drive, Dropbox, etc.)
- Other apps that can receive PDFs

**Given** the user selects an app to share  
**When** the selection is made  
**Then** the PDF should be shared to that app  
**And** the user should be able to complete the action

**Given** sharing is cancelled  
**When** the user closes the share sheet  
**Then** return to Settings page  
**And** PDF should remain saved in Downloads

**Given** sharing fails  
**When** an error occurs  
**Then** an error message should display  
**And** allow user to retry

### Technical Notes
- Use `share_plus` package for Flutter
- Share method: `Share.shareXFiles([XFile(pdfPath)])`
- Share text: "Here's my Bidget budget report for [cycle]"
- MIME type: application/pdf
- Ensure file exists before sharing
- Handle share result (success/cancelled)

### Dependencies
- REP-001 (PDF must be generated first)

### Definition of Done
- [ ] Code implemented and reviewed
- [ ] Share button appears after generation
- [ ] System share sheet opens
- [ ] PDF attached correctly
- [ ] Can share to various apps
- [ ] Share text included
- [ ] Error handling works
- [ ] Tested on multiple apps
- [ ] Unit tests written
- [ ] Integration tests passed
