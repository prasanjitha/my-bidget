## Story CAT-001: Create New Category

**Story ID**: CAT-001  
**Story Title**: Add Custom Expense Category  
**Priority**: Must Have (P0)  
**Story Points**: 3

### User Story
**As a** user  
**I want to** create custom expense categories  
**So that** I can organize my expenses according to my needs

### Acceptance Criteria

**Given** the user wants to create a category  
**When** the "Add New Category" button is tapped (from Home or Expenses page)  
**Then** a popup dialog should open with:
- Category name input field
- Save button
- Cancel button

**Given** the user enters a valid category name  
**When** the Save button is pressed  
**Then** the category should be saved to Firebase `categories` collection with:
- userId
- name
- isDefault: false
- createdAt timestamp
- updatedAt timestamp
**And** the popup should close  
**And** the new category should appear in relevant lists

**Given** the user enters a name that already exists  
**When** Save is attempted  
**Then** an error should display: "Category already exists"  
**And** save should be prevented

**Given** the category name is empty or only whitespace  
**When** Save is attempted  
**Then** validation error should display: "Category name is required"

**Given** the category name exceeds 50 characters  
**When** Save is attempted  
**Then** validation error should display: "Category name too long (max 50 characters)"

**Given** the user enters special characters  
**When** the name is entered  
**Then** alphanumeric and spaces should be allowed  
**And** special characters like @#$%^ should be allowed

### Technical Notes
- Firestore collection: `categories`
- Document ID: Auto-generated by Firestore
- Duplicate check: Query where userId = X AND name = Y (case-insensitive)
- Trim whitespace before saving
- Max length: 50 characters
- Min length: 1 character

### Dependencies
- None (foundational story)

### Definition of Done
- [ ] Code implemented and reviewed
- [ ] Popup displays correctly
- [ ] Category name validation works
- [ ] Duplicate check functional
- [ ] Save creates category in Firebase
- [ ] Category appears in lists
- [ ] Error messages display correctly
- [ ] Unit tests written
- [ ] Integration tests passed

**Priority**: Must Have (P0)  
**Story Points**: 2

### User Story

**So that** I can see what categories I have available

- Default categories (Food, Rent)

## Story CAT-003: Edit Category

**Story ID**: CAT-003  
**Story Title**: Update Existing Category Name  
**Priority**: Should Have (P1)  
**Story Points**: 2

### User Story
**As a** user  
**I want to** edit a category name  
**So that** I can correct typos or rename categories

### Acceptance Criteria

**Given** the user long-presses a category  
**When** the long-press is detected  
**Then** an edit popup should open with:
- Category name input (pre-filled with current name)
- Save button
- Cancel button

**Given** the user is viewing the category filter dropdown  
**When** an edit icon next to a category is tapped  
**Then** the edit popup should open

**Given** the user modifies the category name  
**When** the Save button is pressed  
**Then** the category should be updated in Firebase  
**And** updatedAt timestamp should be set  
**And** all expenses with this categoryId should reflect the new name

**Given** the new name is a duplicate  
**When** Save is attempted  
**Then** an error should display: "Category name already exists"

**Given** the user tries to edit a default category  
**When** edit is attempted  
**Then** editing should be allowed  
**And** isDefault flag should remain true

### Technical Notes
- Update Firestore: `categories/{categoryId}`
- Real-time update: All UI components showing category name
- No need to update expenses collection (use join on categoryId)
- Allow editing default categories
- Validation same as create

### Dependencies
- CAT-002 (Category list to edit from)

## Story CAT-004: Delete Category

**Story ID**: CAT-004  
**Story Title**: Remove Unused Category  
**Priority**: Should Have (P1)  
**Story Points**: 3

### User Story
**As a** user  
**I want to** delete categories I no longer use  
**So that** my category list stays organized

### Acceptance Criteria

**Given** the user swipes a category or taps delete icon  
**When** the delete action is triggered  
**Then** the app should check if the category has expenses in current budget cycle

**Given** the category has expenses in the current cycle  
**When** delete is attempted  
**Then** an error dialog should display: "Cannot delete category with expenses in current budget cycle"  
**And** deletion should be prevented

**Given** the category has no expenses in current cycle  
**When** delete is attempted  
**Then** a confirmation dialog should appear:
- Message: "Are you sure you want to delete '[Category Name]'?"
- Cancel button
- Delete button (red)

**Given** the user confirms deletion  
**When** the Delete button is pressed  
**Then** the category should be deleted from Firebase  
**And** the category should be removed from all lists  
**And** success message: "Category deleted"

**Given** the category has expenses in past cycles  
**When** deletion occurs  
**Then** past expenses should retain the categoryId reference  
**And** display category name from cached/historical data

**Given** the user tries to delete a default category  
**When** delete is attempted  
**Then** deletion should be allowed if no expenses in current cycle

### Technical Notes
- Check expenses: Query where categoryId = X AND budgetCycle = current
- Soft delete option: Set isActive: false instead of hard delete
- Historical data: Consider keeping category reference for reporting
- Default categories: Can be deleted but should warn user

### Dependencies
- CAT-002 (Category list to delete from)
- EXP-002 (Expense checking)

### Definition of Done
- [ ] Code implemented and reviewed
- [ ] Delete action triggers check
- [ ] Prevention works for categories with current expenses
- [ ] Confirmation dialog displays
- [ ] Delete removes from Firebase
- [ ] UI updates immediately
- [ ] Historical expenses unaffected
- [ ] Success/error messages display
- [ ] Unit tests written
- [ ] Integration tests passed

## Story CAT-005: Initialize Default Categories

**Story ID**: CAT-005  
**Story Title**: Create Default Categories on First Launch  
**Priority**: Must Have (P0)  
**Story Points**: 2

### User Story
**As a** new user  
**I want** default categories to be available immediately  
**So that** I can start tracking expenses without setup

### Acceptance Criteria

**Given** the app is launched for the first time  
**When** the user completes biometric enrollment  
**Then** two default categories should be created in Firebase:
- Food (isDefault: true)
- Rent (isDefault: true)

**Given** default categories already exist  
**When** the app is launched  
**Then** no duplicate default categories should be created

**Given** the default categories creation fails  
**When** Firebase write fails  
**Then** the app should retry up to 3 times  
**And** if still failing, allow app to continue  
**And** user can manually create categories

### Technical Notes
- Create during user initialization after AUTH-001
- Firestore batch write for both categories
- Check if already exist: Query where userId = X AND isDefault = true
- Categories:
  - Food: {name: "Food", isDefault: true, userId: X, createdAt, updatedAt}
  - Rent: {name: "Rent", isDefault: true, userId: X, createdAt, updatedAt}
- Retry logic: Exponential backoff

### Dependencies
- AUTH-001 (User must be authenticated first)
- CAT-001 (Category creation logic)

### Definition of Done
- [ ] Code implemented and reviewed
- [ ] Default categories created on first launch
- [ ] Duplicate prevention works
- [ ] Retry logic implemented
- [ ] App continues on failure
- [ ] Batch write used
- [ ] Unit tests written
- [ ] Integration tests passed
